آموزش گیت و گیت هاب (Git & Github)

گیت (Git) یک سیستم کنترل ورژن است که با آن می‌توانید تغییرات اعمال شده در فایل‌ها را ساده‌تر پیگیری کنید.

branch

ساخت branch با دستور زیر یک branch ایجاد می کنیم git branch <name> نشان دادن branch فعلی git branch تعویض branch git switch <name of branch> هر تغییراتی در هر برنچی صورت گیرد فقط بر روی همان برنچ اعمال می شود یک برنچ ایجاد و روی آن سویچ می کند git switch -c <name> تعویض نام برنچ بر روی برنچ مورد نظر سویچ میکنیم git branch -M <new name>

git add & git commit

git add با استفاده از cmd وارد دایرکتوری مورد نظر می شویم git init git status برای رهگیری و تِرَک فایل ها در یک دایرکتوری از دستورات زیر استفاده می کنیم git add <file> یا git add . یا git add <file1> <file2> برای عدم رهگیری و بازگشت به حالت قبل git rm --cached <file> git commit با استفاده از مسیج کامیت ها را رهگیری میکنیم git commit -m "<massage>" برای بار اول باید یوزر و ایمیل را وارد کنیم...

git diff

git diff مقایسه دایرکتوری و استیج git diff git diff –staged مقایسه آخرین کامیت و استیج git diff --staged git diff HEAD مقایسه آخرین کامیت و دایرکتوری git diff HEAD git diff <hash_comit_1>..<hash_commit_2> مقایسه دو کامیت git diff <hash_comit_1>..<hash_commit_2> برای مشاهده هش: git log --oneline git diff <branch_1>..<branch_2> مقایسه دو برنچ با هم git diff <branch_1>..<branch_2>

git ignore

git ignore برای نادیده گرفتن تعدادی از فایل های در git status از .gitignore استفاده میکنیم در دایرکتوری مورد نظر یک فایل با نام .gitignore ایجاد میکنیم و تمامی فایل هایی که می خواهیم توسط گیت نادیده گرفته شوند را در آن می نویسیم .gitignore file4.txt فایل بالا در git status نشان داده نمی شود

merge branch

merge brach بر روی برنچ master می رویم سپس دستور زیر: git merge <name of branch> Conflict وارد فایل مورد نظر در ادیتور شده تداخلات نشان داده شده را رفع می کنیم و مجددا: git add . git commit -m "conflict fix"

remote & push

remote git remote add <name(origin)> <url> name: یک لیبل بصورت قراردادی است و هر اسمی می تواند باشد دستور زیر لیبل را نشان میدهد git remote نشان دادن remote url git remote -v تغییر url git remote set-url <name(origin)> <new url> Remote branch زمانی که یک پروژه را clone می کنیم فقط برنچ اصلی را نشان میدهد برای دسترسی به remote branch ها از دستور زیر استفاده می کنیم git switch <branch name> ابتدا پروژه را کلون می کنیم git clone <url> مشاهده برنچ ها git branch مشاهده ریموت برنچ ها git branch -r دسترسی به سایر برنچ ها git switch <branch name> push git push -u <remote name> <branch name>

rename & delete branch

Rename branch ابتدا با دستور git switch بر روی برنچ مورد نظر رفته سپس با دستور زیر برنچ را تغییرنام می دهیم git branch -m <renamed name> Delete branch ابتدا بر روی برنچی دیگر می رویم git branch -D <name of branch>

restore

restore برای حذف تغییرات جدیدی که در دایرکتوری ایجاد کرده ایم git restore <file> restore برای unstage کردن git restore --staged <file> بازگشت به یک کامیت بدون تغییر هد git restore --source <hash> <file>

revert

revert بازگشت به یک کامیت انجام تغییرات در آن کامیت و حل conflict ها مجددا برگشت به کامیت آخر git revert <hash> git add . git revert --continue تفاوت با دستور reset دستور ریست کامیت ها را پاک می کند اما revert آنها را نگه میدارد
